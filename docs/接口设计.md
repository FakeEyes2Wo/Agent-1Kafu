# 接口设计

## 1. 基础信息

系统提供一个核心客服接口：

```http
POST /chat
```

服务框架：

- FastAPI
- LangGraph Agent
- LangChain OpenAI 兼容模型

启动脚本：

```powershell
uv run python scripts/run_api.py
```

默认监听地址由 `.env` 控制：

```env
APP_HOST=0.0.0.0
APP_PORT=8000
```

## 2. 认证方式

所有 `/chat` 请求必须携带 Bearer Token：

```http
Authorization: Bearer {KAFU_API_TOKEN}
```

`KAFU_API_TOKEN` 来自 `.env`：

```env
KAFU_API_TOKEN=change-me
```

认证失败返回 HTTP 401。

## 3. 请求格式

请求头：

```http
Content-Type: application/json
Authorization: Bearer change-me
```

请求体：

```json
{
  "question": "我想更换健身追踪器的表带，有其他尺寸可选吗？",
  "images": [],
  "session_id": "kf_session_001",
  "stream": false
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `question` | string | 是 | 用户客服问题，不能为空 |
| `images` | list[string] | 否 | Base64 图片列表，默认空列表 |
| `session_id` | string | 否 | 会话 ID，不传则自动生成 |
| `stream` | boolean | 否 | 第一版接受该字段，但统一返回非流式结果 |

图片建议格式：

```text
data:image/png;base64,{base64内容}
```

## 4. 成功响应

响应格式：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "answer": "您好，健身追踪器出厂配有小号表带，包装盒内另附大号底部表带。顶部和底部表带都可以与配件表带互换，表带尺寸请参考手册中的表带尺寸说明。<PIC>,[\"Manual16_51\"]",
    "session_id": "kf_session_001",
    "timestamp": 1741008000
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | int | 业务状态码，成功为 0 |
| `msg` | string | 状态描述 |
| `data.answer` | string | Agent 生成并后处理后的客服答案；图片引用以正文 `<PIC>` 加末尾图片 ID 列表呈现 |
| `data.session_id` | string | 当前会话 ID |
| `data.timestamp` | int | 秒级 Unix 时间戳 |

## 5. 错误响应

认证失败：

```json
{
  "detail": "invalid token"
}
```

HTTP 状态码：

```text
401
```

问题为空：

```json
{
  "detail": "question is required"
}
```

HTTP 状态码：

```text
400
```

请求体字段类型错误时，由 FastAPI 返回 422。

## 6. 调用示例

PowerShell 示例：

```powershell
$body = @{
  question = "物流一直显示待揽收，是什么原因？"
  images = @()
  session_id = "kf_session_001"
  stream = $false
} | ConvertTo-Json

Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/chat" `
  -Method Post `
  -Headers @{ Authorization = "Bearer change-me" } `
  -ContentType "application/json" `
  -Body $body
```

## 7. 与比赛提交的关系

比赛提交文件不通过 API 间接生成，而是直接运行脚本：

```powershell
uv run python scripts/generate_submission.py
```

脚本读取：

```text
data/question_public.csv
```

参考：

```text
data/submission_example.csv
```

输出：

```text
submission.csv
```

提交文件字段固定为：

```csv
id,ret
```

其中：

- `id` 与 `question_public.csv` 保持一致。
- `ret` 为 Agent 生成的最终客服回答。
- 如果某条生成失败，脚本会中断并暴露错误，避免写入非模型生成的模板回答。

## 8. 接口设计取舍

第一版接口保持简单：

- 只提供 `/chat` 一个核心接口。
- 不实现 SSE 流式响应。
- 不暴露索引构建接口。
- 不暴露调试检索接口。
- 不在 API 请求路径中执行长时间重建索引任务。

这样可以降低比赛部署和评测风险。如果后续需要在线调试，可再增加 `/debug/retrieve` 或 `/admin/rebuild-index` 等内部接口。
